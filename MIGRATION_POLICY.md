# Migration Policy

- Do not copy code blindly.
- Harvest only after read-only audit.
- Every migrated feature must document source repo, reason, tests, owner module, and rollback path.
- Every migrated feature must serve Case → Evidence → Map → Action → Learning.
- Features that do not serve the daily loop are PAUSED.
- Public repo contains only code, docs, schema, and synthetic examples.
- Runtime files, local cases, raw office docs, screenshots, logs, tokens, and export packs stay out of git.
